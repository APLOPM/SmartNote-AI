# UI/UX Specification: หน้าจอแก้ไขโน้ตหลัก (Main Note Editor)

**เวอร์ชัน:** 1.0
**วันที่:** 4 มีนาคม 2026

---

### 1. ภาพรวม (Overview)

หน้าจอ "Main Note Editor" คือพื้นที่ทำงานหลักที่ผู้ใช้จะใช้เวลาส่วนใหญ่อยู่ที่นี่ เป็นศูนย์กลางของการสร้างและแก้ไขเนื้อหา, การเรียกใช้ AI, และการจัดการโน้ต การออกแบบมุ่งเน้นที่ความเรียบง่าย, ลดสิ่งรบกวน (Distraction-Free) แต่ยังคงความสามารถในการเข้าถึงเครื่องมือ AI ที่ทรงพลังได้อย่างรวดเร็ว

### 2. โครงสร้างและเลย์เอาต์ (Layout)

หน้าจอถูกแบ่งออกเป็น 3 ส่วนหลัก:

1.  **Top Bar:** แสดงชื่อโน้ต, breadcrumbs, และเมนูสำหรับการแชร์หรือ Export
2.  **Content Area:** พื้นที่สำหรับพิมพ์และแก้ไขเนื้อหา รองรับ Rich Text, Markdown, และการฝังองค์ประกอบต่างๆ (รูปภาพ, ไฟล์)
3.  **AI Command Button (FAB):** ปุ่มหลักสำหรับเรียก AI Agent จะลอยอยู่มุมขวาล่างของจอ

### 3. รูปแบบการปฏิสัมพันธ์กับ AI (AI Interaction Patterns)

เราจะใช้ 3 รูปแบบหลักตามที่ระบุใน `AI UX Design System`:

#### 3.1. Agent-Level Task (Command Palette)

เป็นการเรียกใช้ Agent สำหรับงานที่ซับซ้อนและมีหลายขั้นตอน

-   **Trigger:** ผู้ใช้กดปุ่ม **✨ (FAB)** ที่มุมขวาล่าง
-   **UI Flow:**
    1.  เมื่อกดปุ่ม, หน้าต่าง Modal จะปรากฏขึ้นกลางจอ พร้อม input field และคำแนะนำ เช่น `"สรุปโน้ตนี้เป็น Bullet Point"` หรือ `"สร้าง Presentation จากเนื้อหานี้"`
    2.  ผู้ใช้พิมพ์เป้าหมาย (Goal) ที่ต้องการลงไป
    3.  AI Agent จะแสดงแผนการทำงาน (Planning) เพื่อให้ผู้ใช้ยืนยัน เช่น `"ฉันจะอ่านโน้ต, สรุปประเด็นสำคัญ, และสร้างไฟล์ PPT ให้คุณ. [เริ่มทำงาน] [แก้ไข]"`
    4.  เมื่อเริ่มทำงาน, จะมี Status Indicator แสดงความคืบหน้า เช่น `"กำลังสร้างสไลด์ (3/10)..."`
    5.  เมื่องานเสร็จสิ้น, ผลลัพธ์จะถูกแสดงในรูปแบบที่เหมาะสม (เช่น ลิงก์ดาวน์โหลดไฟล์, หรือเนื้อหาที่ถูกเพิ่มเข้ามาในโน้ต)

#### 3.2. In-line AI Assistance (Slash Commands)

สำหรับฟังก์ชัน AI ที่ทำงานอย่างรวดเร็วภายในเนื้อหาโดยตรง

-   **Trigger:** ผู้ใช้พิมพ์เครื่องหมาย `/` ในพื้นที่ Content Area
-   **UI Flow:**
    1.  เมนูลอย (Floating Menu) จะปรากฏขึ้นข้างๆ cursor
    2.  ในเมนูจะแสดงรายการคำสั่ง AI ที่ใช้บ่อย เช่น:
        -   `/summarize` (สรุปย่อ)
        -   `/translate` (แปลภาษา)
        -   `/fix-grammar` (แก้ไวยากรณ์)
        -   `/ask` (ถามคำถามเกี่ยวกับเนื้อหานี้)
    3.  เมื่อเลือกคำสั่ง, AI จะประมวลผลและแทรกผลลัพธ์ลงในตำแหน่งของ cursor ทันที โดยแสดงเป็น "AI Block"

#### 3.3. Contextual AI (Text Selection Toolbar)

เมื่อผู้ใช้ต้องการให้ AI ทำงานกับข้อความส่วนที่เลือกไว้

-   **Trigger:** ผู้ใช้เลือก (highlight) ข้อความส่วนหนึ่ง
-   **UI Flow:**
    1.  Toolbar ขนาดเล็กจะปรากฏขึ้นเหนือข้อความที่เลือก
    2.  Toolbar นี้จะมีปุ่ม format พื้นฐาน (Bold, Italic) และตามด้วยไอคอน **✨**
    3.  เมื่อกดไอคอน ✨, เมนู AI จะปรากฏขึ้นพร้อมคำสั่งที่เกี่ยวข้องกับบริบทนั้นๆ เช่น "ขยายความ", "หาข้อมูลเพิ่มเติมเกี่ยวกับ...", "เปลี่ยนโทนเป็นทางการ"
    4.  ผลลัพธ์จะแทนที่ข้อความเดิม (พร้อมตัวเลือก Undo) หรือแสดงใน AI Block ด้านล่าง

### 4. การแสดงผลลัพธ์จาก AI (AI-Generated Content)

ผลลัพธ์ทั้งหมดที่สร้างโดย AI จะถูกแสดงใน "AI Block" ที่ออกแบบไว้ใน Design System เสมอ:

-   มีพื้นหลังสีแตกต่างเล็กน้อยเพื่อแยกจากเนื้อหาที่ผู้ใช้เขียนเอง
-   มีไอคอน ✨ กำกับที่มุมบนซ้าย
-   มีปุ่ม Action ที่ท้าย Block เสมอ: `[ ยอมรับ ]`, `[ สร้างใหม่ ]`, `[ ยกเลิก ]` เพื่อให้ผู้ใช้ควบคุมผลลัพธ์ได้อย่างสมบูรณ์
